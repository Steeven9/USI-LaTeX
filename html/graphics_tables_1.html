Building a simple table
In order to create the table, it is necessary to begin the table enviroment, using the comand \begin{table}. This is not sufficient to begin the table, to create the table we use the tabular enviroment, which generates the table.
General command is \begin{tabular}[pos]{spec}.
The pos stays for position of the table in the page. This is explained in the section Positioning: Position of the table in the page. In the position of spec stay the comands, used to determine the number of columns, their width and their alignment. The tables in LaTeX are deteminated by the number of spec attributes.
  • l - the content of the table will be aligned to the left side
  • c - the content of the table will be aligned to the center side
  • r - the content of the table will be aligned to the rigth side

code

\documentclass{article}
\begin{document}
\begin{table}
\begin{tabular}{lcr}
table & long table & not so long table \\
1 & 123456789000 & 3.14 \\
\end{tabular}
\end{table}
\end{document}


In the example above, the number of columns is 3: the content of first is aligned on the left, of the second one in the center and of the third on the left.
The size of the cell depends on the size of the content in the cell, so the biggest content will set the size. Despite, it is possible to determine the the size of column and row (see chapter Special columns and Spacing).
  • \\ - signature is used to separate rows.
  • & - signature is used to separate columns.
In all the next examples, the code of \begin{document}, \begin{table}, \end{table}, \end{document} will not be shown, in order to dedicate more attention to the importantdetails. Although these elements are necessary for the code in LaTeX. The commands will be shown only in case are needed for demostration.

code

\begin{table}
\begin{tabular}{rcr}
example & & example\\
long example & not so long example & example\\
\end{tabular}
\caption{Example of the table with caption and one single empty column}
\end{table}


In this example, it is shown the \caption{} command. It is used to write a small description of the table. In the exxample, the caption is aligned on the left. For more details see Positioning: Alignment.
Also, it is possible to see the empty column. In order to create it, the input for the cell is empty, in other words place two & symbols: example & & example without putting nothing inside.
Adding lines
In this chapter is shown how to add the lines, both vertical and horizontal to the table.
In order to add vertical line, the symbol | is added in the tabular enviroment, e.g. \begin{tabular}{|r|c|r}. If the line must be on the right of the columns, so the | symbol is putted on the right, vice versa for the left. Note, that the vertical line is propagated over all the table. In order to get a table with vertical lines and one cell without vertical lines, see Cell spanning: Multicolumns.
While, for the horizontal line the procedure is different: the command \hline must be added after or before every row. Also this command designes the line over all the width of the table.
Using \cline{i-j}, it is possible to draw a horizontal line only for deteminated columns.

code

\begin{tabular}{|r|c|r}\hline
example & & examplemd\\ \hline
long example & not so long example & example\\ /cline{2-3}
\end{tabular}


In this example were added 3 vertical lines and 2 horizontal. Also, here is shown the \cline command. In this case, the code says to draw a horizontal line from cell 2 to cell 3.
Special columns
In this section the new types of columns are introduced.
  • p{width} - creates a column of maximum width, with the content aligned vertically at the top, while hotizontally the content is aligned at the left
  • m{width} - creates a column of maximum width, with the content of all other cells aligned vertically at the middle, while hotizontally the content is aligned at the left; use the \usepackage{array}
  • b{width} - creates a column of maximum width, with the content of all other cells aligned vertically at the bottom, while hotizontally the content is aligned at the left; use the \usepackage{array}

    • code

    \begin{tabular}{|r|p{2cm}|r|}\hline
    example & & example for cell\\ \hline
    long example & not so long example not very long& example\\\hline
    \end{tabular}

    \begin{tabular}{|r|m{2cm}|r|}\hline
    example & & example for cell\\ \hline
    long example & not so long example not very long& example\\\hline
    \end{tabular}

    \begin{tabular}{|r|b{2cm}|r|}\hline
    example & & example for cell\\ \hline
    long example & not so long example not very long& example\\\hline
    \end{tabular}


    In the first table the p is used. In fact, all the content of other cell is aligned at the top. Command m alignes the content at the middle(second table) and b at the bottom(third table).
Spacing
This chapter speaks about the sapce between columns and rows.
First, it is more easy to introduce the command \setlength{\tabcolsep}{width}, which modifies space between columns. The default width between the columns is 6pt.

code

\setlength{\tabcolsep}{1pt}
\begin{table}
\begin{tabular}{|r|m{2cm}|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\\hline
example& table & tables\\
\end{tabular}
\end{table}

\setlength{\tabcolsep}{7pt}
\begin{table}
\begin{tabular}{|r|m{2cm}|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\\hline
example& table & tables\\
\end{tabular}
\end{table}

\setlength{\tabcolsep}{25pt}
\begin{table}
\begin{tabular}{|r|m{2cm}|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\\hline
example& table & tables\\
\end{tabular}
\end{table}


The first table has 1pt as the widht between columns, in fact it is very tight. The second one has almost normal width and the third one has very large width (25pt).
Meanwhile for spacing between the rows, exists different ways. One of the method is to redefine the space between the rows (\arraystretch) by using \renewcommand{\arraystretch}{height}. The default spacing is 1.0.

code

\begin{table}
\begin{tabular}{|r|c|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\\hline
\end{tabular}
\end{table}

\begin{table}
\renewcommand{\arraystretch}{1.5}
\begin{tabular}{|r|c|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\\hline
\end{tabular}
\end{table}

\begin{table}
\renewcommand{\arraystretch}{6}
\begin{tabular}{|r|c|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\\hline
\end{tabular}
\end{table}


The first table is default. The second one has a little spacing between the rows. While the last one uses large spacing. As you can see the more space there is the more the missalignment is visible. In order to fix this go to the section Cell spanning: alignment.
Another way to add space between the rows is to use \noalign{\smallskip} after or before the \hline and \cline commands. It is quiete useless, except like in the examples below. Instead insering [height] is more practical.

code
\begin{table}
\begin{tabular}{|r|c|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\ \hline
example& table & tables\\
\end{tabular}
\end{table} \begin{table}
\begin{tabular}{|r|c|r|}\hline
example & & example for cell\\ \hline \noalign{\bigskip} \hline
long example & not so long example not very long& example\\ \noalign{\bigskip}
example& table & tables\\
\end{tabular}
\end{table} \begin{table}
\begin{tabular}{rcr}\hline
example & & example for cell\\ \hline \noalign{\bigskip} \hline
long example & not so long example not very long& example\\ \noalign{\smallskip}
example& table & tables\\
\end{tabular}
\end{table} \begin{table}
\begin{tabular}{|rcr|}\hline
example & & example for cell\\ [0.5cm] \hline
long example & not so long example not very long& example\\ [1.5cm] \hline
example& table & tables\\
\end{tabular}
\end{table}

The first table is default. In the second one the command \noalign{\bigskip} is used. As you can see, it creates the space between rows but also between the lines, so it is good to use double/hline or to avoid the vertical lines, like shown in third table.
The fourth table uses [height] to create space between rows. This command is more practical and it does not influence the lines.
Borders
In order to create beautiful border, the LaTeX needs some packages array,booktabs,arydshln,xcolor.

code

\usepackage{array,booktabs,arydshln,xcolor}
\newcommand\VRule[1][\arrayrulewidth]{\vrule width #1}
\begin{tabular}{!{\VRule[2pt]}r!{\VRule}c!{\color{green}\VRule[3pt]}c!{\VRule}} \hline
Column 1 & Column 2 & column3 \\ \specialrule{2pt}{0pt}{0pt}
second row & & \\ \hdashline
third row & & \\ \specialrule{4pt}{0pt}{0pt}
example & 1234 & super example\\ \specialrule{1pt}{3pt}{0pt}
\end{tabular}


Firstly, we must add 4 packages \usepackage{array,booktabs,arydshln,xcolor} and the define new command (VRule). Inside the tabular enviroment, the inputs (c, l, r) must be accompanied by ! symbol. The new command VRule[width] is used to generate the vertical lines of certain width. Also, it is possible to change the color by adding \color{color} before the VRule[width] command.
The \hdashline command, which adds the dashed horizontal line. At the end the \specialrule{pt}{pt}{pt} adds the editable horizontal line: the first input is the width of the line, the second one is the vertical positioning and the last one is horizontal.
Size of the table
In this last chapter is explained how to manage the size of the table. The most common way is to use package graphicx and the command \resizebox{width}{height}{object}.

code

\usepackage{graphicx}
\begin{document}
\begin{table}
\resizebox{1.5\textwidth}{!}{\begin{minipage}{\textwidth}
\begin{tabular}{|r|c|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\ \hline
example& table & tables\\
\end{tabular}
\end{minipage}}
\end{table}

\resizebox{12cm}{!}{\begin{minipage}{\textwidth}
\begin{tabular}{|r|c|r|}\hline
example & & example for cell\\ \hline
long example & not so long example not very long& example\\ \hline
example& table & tables\\
\end{tabular}
\end{minipage}}
\end{table}


In the first table, we imposed the % of the width of the text area. While in the second one it is used a fix value in cm.
